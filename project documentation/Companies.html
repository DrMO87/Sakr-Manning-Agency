Module Overview

Companies API is a Django REST Framework module for managing company/employer information in maritime crew management systems.

Core Capabilities

  • Company Management: CRUD operations for company records
  • Company Categorization: Type-based classification (Shipping, Cruise, Cargo, etc.)
  • Hourly Rate Tracking: Built-in field for payroll calculations
  • Position Management: Track open positions for recruitment
  • Status Tracking: Active/Inactive lifecycle management
  • RESTful API: Full CRUD with DRF ModelViewSet
Tech Stack: Django 4+, Django REST Framework, Python 3.8+

Installation & Setup

Add to Installed Apps

INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    # ...
    'rest_framework',
    'companies',  # Add this
]

Include URLs

# In main urls.py
urlpatterns = [
    # ...
    path('api/companies/', include('companies.urls')),
]

Run Migrations

python manage.py makemigrations companies
python manage.py migrate
The module is now accessible at /api/companies/ and admin panel at /admin/companies/company/

API Endpoints

Base URL: /api/companies/

Standard CRUD Operations

Method Endpoint Description Auth
GET / List all companies (paginated) Authenticated
POST / Create new company Authenticated
GET /{id}/ Retrieve company details Authenticated
PUT /{id}/ Update entire company record Authenticated
PATCH /{id}/ Partial update company record Authenticated
DELETE /{id}/ Delete company Authenticated

Serialized Fields

All model fields are serialized, including id, created_at, and updated_at.

The API supports pagination, filtering, and ordering by default through DRF's ModelViewSet.

Key Features

Industry-Specific Design

  • Pre-defined company types tailored for maritime industry
  • Status tracking for active/inactive partnerships

Payroll Integration

  • hourly_rate field for wage calculations
  • Used by Finance module to compute daily rates
  • Decimal precision for accurate financial data

Recruitment Support

  • Track open positions across companies
  • Searchable by company name and email

API-Ready

  • Full CRUD operations via ModelViewSet
  • No additional configuration needed
  • Automatic serialization of all fields

Data Integrity

  • Unique company names enforced at database level
  • Positive integer validation for open positions
  • Automatic timestamps for audit trail

Usage Examples

Create New Company

POST /api/companies/
Content-Type: application/json
Authorization: Token your-token-here

{
  "company_name": "Oceanic Shipping Ltd.",
  "company_type": "Shipping",
  "status": "Active",
  "contact_email": "hr@oceanic-shipping.com",
  "hourly_rate": 20.50,
  "open_positions": 15
}

Response

{
  "id": 7,
  "company_name": "Oceanic Shipping Ltd.",
  "company_type": "Shipping",
  "status": "Active",
  "open_positions": 15,
  "contact_email": "hr@oceanic-shipping.com",
  "hourly_rate": "20.50",
  "created_at": "2025-11-10T10:30:00Z",
  "updated_at": "2025-11-10T10:30:00Z"
}

List with Filtering

GET /api/companies/?status=Active&company_type=Shipping
Authorization: Token your-token-here

Update Company

PATCH /api/companies/7/
Content-Type: application/json
Authorization: Token your-token-here

{
  "open_positions": 12,
  "status": "Inactive"
}

Delete Company

DELETE /api/companies/7/
Authorization: Token your-token-here
