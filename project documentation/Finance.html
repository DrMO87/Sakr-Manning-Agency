Module Overview

Finance API is a Django REST Framework module for tracking financial records, calculating payments, and managing user-company financial relationships.

Core Capabilities

  • Financial Record Tracking: Track work periods and payment calculations
  • Automatic Calculations: Days worked, daily rates, and total payments
  • Company Integration: Links users with companies for payroll processing
  • Preview Calculations: Calculate payments without saving records
  • Admin Management: Comprehensive admin interface with calculated fields
Tech Stack: Django 4+, Django REST Framework, Python 3.8+

Installation & Setup

Add to Installed Apps

INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    # ...
    'rest_framework',
    'finance',  # Add this
    'api',      # Required for Users model
    'companies', # Required for Company model
]

Run Migrations

python manage.py makemigrations finance
python manage.py migrate
Dependencies: This module requires the api and companies apps to be installed.

Models Documentation

FinanceRecord

Core model for tracking financial records and calculating payments.

Field Type Description
userForeignKey(Users)Related seafarer/worker
companyForeignKey(Company)Employer company
start_dateDateFieldWork period start date
end_dateDateFieldWork period end date
created_atDateTimeFieldAuto-set on creation
updated_atDateTimeFieldAuto-set on update

Calculated Properties

Property Description Calculation
total_days Total days worked (end_date - start_date).days + 1
daily_rate Daily wage rate company.hourly_rate * 8 (assumes 8 hrs/day)
total_money Total payment amount total_days * daily_rate
String Representation: "{first_name} @ {company_name} ({start_date} → {end_date})"

API Endpoints

Base URL: /api/finance-records/

Standard CRUD Operations

Method Endpoint Description Auth
GET / List all finance records (paginated) Authenticated
POST / Create new finance record Authenticated
GET /{id}/ Get specific record details Authenticated
PUT /{id}/ Update entire record Authenticated
PATCH /{id}/ Partial update record Authenticated
DELETE /{id}/ Delete finance record Authenticated

Custom Action

Method Endpoint Description
POST /calculate/ Calculate payment without saving to database

Calculate Endpoint Details

This endpoint performs all calculations without persisting data to the database.

Request Example:

POST /api/finance-records/calculate/
Content-Type: application/json

{
  "user": 1,
  "company": 2,
  "start_date": "2025-09-01",
  "end_date": "2025-09-10"
}

Response Example:

{
  "total_days": 10,
  "daily_rate": 160.00,
  "total_money": 1600.00
}

Django Admin Interface

URL: /admin/finance/financerecord/

List Display

The admin interface shows calculated fields directly in the list view:

Admin Column Source Description
get_total_daystotal_days propertyTotal days worked
get_daily_ratedaily_rate propertyDaily wage rate
get_total_moneytotal_money propertyTotal payment amount

Filters & Search

  • Filters: company, user, start_date, end_date
  • Search: user first_name, last_name, company company_name
All calculated fields appear as read-only fields in the admin detail view.

Key Features

Smart Calculations

  • Automatic Day Calculation: Includes both start and end dates
  • Dynamic Rate Lookup: Pulls hourly_rate from Company model
  • Zero Dependencies: Works even if company rate is not set

Performance Optimized

  • Properties: Calculated on-demand, no database queries
  • Serializer: Uses ReadOnlyField for calculated values
  • Admin: Custom methods for list display optimization

Data Integrity

  • Date Validation: Ensures end_date >= start_date (via model validation)
  • Cascading Deletes: Records deleted when user or company is removed
  • Audit Trail: created_at and updated_at timestamps

Usage Examples

Create Finance Record

POST /api/finance-records/
Content-Type: application/json
Authorization: Token your-token-here

{
  "user": 5,
  "company": 2,
  "start_date": "2025-09-15",
  "end_date": "2025-09-30"
}

Response

{
  "id": 12,
  "user": 5,
  "company": 2,
  "start_date": "2025-09-15",
  "end_date": "2025-09-30",
  "created_at": "2025-11-10T10:30:00Z",
  "updated_at": "2025-11-10T10:30:00Z",
  "total_days": 16,
  "daily_rate": 160.00,
  "total_money": 2560.00
}

Preview Calculation Without Saving

POST /api/finance-records/calculate/
Content-Type: application/json
Authorization: Token your-token-here

{
  "user": 3,
  "company": 1,
  "start_date": "2025-10-01",
  "end_date": "2025-10-31"
}

Calculation Response

{
  "total_days": 31,
  "daily_rate": 180.00,
  "total_money": 5580.00
}

Update Record

PATCH /api/finance-records/12/
Content-Type: application/json
Authorization: Token your-token-here

{
  "end_date": "2025-10-05"
}

Delete Record

DELETE /api/finance-records/12/
Authorization: Token your-token-here

Integration Notes

Required Dependencies

App Model Purpose
api Users Worker/employee information
companies Company Employer information and hourly_rate

Assumptions

  • The Company model must have an hourly_rate field
  • Hourly rate is multiplied by 8 to get daily rate
  • Date calculations are inclusive of both start and end dates

Include URLs

# In main urls.py
urlpatterns = [
    # ...
    path('api/', include('finance.urls')),
]
