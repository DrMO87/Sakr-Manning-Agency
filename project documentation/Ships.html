Module Overview

Ships API is a Django REST Framework module for managing vessels, crew assignments, and maritime operations in crew management systems.

Core Capabilities

  • Vessel Management: CRUD operations for ship records with IMO registration
  • Crew Assignment: Many-to-many relationships between ships and seafarers
  • Company Integration: Ships belong to specific maritime companies
  • Technical Specifications: Track tonnage, engine details, build year
  • Flag Registry: Support for vessel flag state registration
  • Dynamic Crew Management: Add/remove crew via dedicated API endpoints
  • DRF Admin Integration: User-friendly crew assignment interface
Tech Stack: Django 4+, Django REST Framework, Python 3.8+

Installation & Setup

Add to Installed Apps

INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    # ...
    'rest_framework',
    'ships',      # Add this
    'api',        # Required for Users model
    'companies',  # Required for Company model
    'core',       # Required for Flag/VesselType models
]

Include URLs

# In main urls.py
urlpatterns = [
    # ...
    path('api/ships/', include('ships.urls')),
]

Run Migrations

python manage.py makemigrations ships
python manage.py migrate
Dependencies: This module requires api, companies, and core apps.

API Endpoints

Base URL: /api/ships/

Standard CRUD Operations

Method Endpoint Description Auth
GET / List all ships (paginated) Authenticated
POST / Create new ship Ship Manager/Admin
GET /{id}/ Retrieve ship details Authenticated
PUT /{id}/ Update entire ship record Ship Manager/Admin
PATCH /{id}/ Partial update ship record Ship Manager/Admin
DELETE /{id}/ Delete ship Ship Manager/Admin

Custom Crew Management Actions

Method Endpoint Description Payload
POST /{id}/assign-user/ Add a user to ship's crew {"user_id": 123}
POST /{id}/unassign-user/ Remove a user from ship's crew {"user_id": 123}
Permission Note: Ship management requires 'Ship Manager' or 'Admin' role permissions.

Serialized Fields

The ShipSerializer provides both read and write capabilities:

  • Read Fields: All model fields plus nested crew details
  • Write Fields: Accepts crew_ids array to bulk assign crew
  • Nested Crew: Returns {id, first_name, last_name, email} for each crew member

Key Features

Comprehensive Vessel Management

  • IMO number validation and uniqueness enforcement
  • Detailed technical specifications (tonnage, engine, builder)
  • Flag state and port of registry tracking

Dynamic Crew Assignment

  • ManyToMany relationship between ships and users
  • Custom API endpoints for adding/removing crew
  • Nested serialization shows crew details in API responses
  • Admin interface with multi-select widget

Company Integration

  • ForeignKey to Company model with CASCADE delete protection
  • Related_name allows reverse queries: company.ships.all()

Role-Based Permissions

  • IsShipManagerOrAdmin permission class
  • Ship managers can manage ships, HR can manage users
  • Admin users have full access

API-Ready

  • Full CRUD with DRF ModelViewSet
  • Custom actions for crew management
  • Nested serializers for complex data structures
  • Automatic timestamp fields

Integration Notes

URL Configuration

Add to your main urls.py:

urlpatterns = [
    # ... other paths
    path('api/', include('ships.urls')),
]

Required Dependencies

App Model Relation Type Purpose
companies Company ForeignKey Ship owner/operator
api Users ManyToMany Crew assignments
core Flag ForeignKey Flag state registration
core VesselType ForeignKey Vessel classification

Database Relations

  • Company → Ships: One-to-Many (One company owns many ships)
  • Ship → Crew: Many-to-Many (Ships have many crew, crew can serve on multiple ships)
  • User → Ships: Reverse Many-to-Many (Access via user.ships.all())

Permission Configuration

The module uses a custom permission class:

# In settings.py or views
REST_FRAMEWORK = {
    'DEFAULT_PERMISSION_CLASSES': [
        'rest_framework.permissions.IsAuthenticated',
    ]
}
Note: The IsShipManagerOrAdmin permission class must be implemented in permissions.py to support role-based access control.

Related Queries

# Get all ships for a company
company.ships.all()

# Get all crew for a ship
ship.crew.all()

# Get all ships a user serves on
user.ships.all()

# Check if user is on a specific ship
ship.crew.filter(id=user_id).exists()
