Module Overview

Tickets & Papers API is a Django REST Framework module for managing travel documents including flight tickets and traveling papers (visas, travel permits) for seafarers.

Core Capabilities

  • Flight Ticket Management: Store and retrieve flight tickets with file uploads
  • Travel Paper Management: Handle visas, travel permits with issue dates
  • File Upload Support: Document storage with automatic URL generation
  • User-Scoped Access: Documents are automatically filtered by user
  • Full CRUD Operations: Complete REST API with ViewSets
  • Multipart Form Support: Handles file uploads via API
Tech Stack: Django 4+, Django REST Framework, Python 3.8+

Installation & Setup

Add to Installed Apps

INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    # ...
    'rest_framework',
    'tickets_papers',  # Add this
    'api',             # Required for Users model
]

Include URLs

# In main urls.py (using nested routing)
from rest_framework_nested import routers

# Main router
router = routers.DefaultRouter()
router.register(r'users', UserViewSet, basename='user')

# Nested router for tickets/papers
users_router = routers.NestedDefaultRouter(router, r'users', lookup='user')
users_router.register(r'tickets', TicketViewSet, basename='user-tickets')
users_router.register(r'traveling-papers', TravelingPaperViewSet, basename='user-papers')

urlpatterns = [
    # ...
    path('api/', include(router.urls)),
    path('api/', include(users_router.urls)),
]
Nested Routing: This module is designed to work with nested URLs under users (e.g., /api/users/{user_id}/tickets/)

Run Migrations

python manage.py makemigrations tickets_papers
python manage.py migrate

API Endpoints

Base URL Pattern: /api/users/{user_id}/

Ticket Endpoints

Method Endpoint Description Auth
GET /tickets/ List all tickets for a specific user Authenticated
POST /tickets/ Upload new ticket (multipart form) Authenticated
GET /tickets/{id}/ Retrieve specific ticket details Authenticated
PUT /tickets/{id}/ Update entire ticket Authenticated
PATCH /tickets/{id}/ Partial update ticket Authenticated
DELETE /tickets/{id}/ Delete ticket Authenticated

PATCH /traveling-papers/{id}/ Partial update paper Authenticated

Serialized Fields

Both serializers provide special handling for file uploads:

  • file: Write-only field for uploading documents
  • file_url: Read-only field returning absolute URL for downloads
  • user: Set automatically from URL, read-only in API

Key Features

File Management

  • Automatic URL Generation: file_url field provides absolute URLs
  • Organized Storage: Files stored in separate tickets/ and traveling_papers/ directories
  • Multipart Form Support: ViewSets configured to handle multipart/form-data

User-Scoped Access

  • Automatic Filtering: Documents filtered by user_id from URL
  • User Assignment: perform_create automatically associates document with user
  • No Manual User ID: User field is read-only in serializer, set from URL path

Nested Routing

  • RESTful Structure: Documents accessed under /users/{id}/ path
  • Clean URLs: Logical hierarchy: User → Documents
  • DRY Principle: No need to pass user_id in request body

Admin Integration

  • Search by User: Search documents by crew member name
  • Date Filtering: Filter by creation/issued dates
  • Quick Access: Direct links from user records in admin

Integration Notes

Nested Router Setup

Complete URL configuration example:

# main urls.py
from django.urls import path, include
from rest_framework_nested import routers
from api.views import UserViewSet
from tickets_papers.views import TicketViewSet, TravelingPaperViewSet

# Main router for users
router = routers.DefaultRouter()
router.register(r'users', UserViewSet, basename='user')

# Nested router for user documents
users_router = routers.NestedDefaultRouter(router, r'users', lookup='user')
users_router.register(r'tickets', TicketViewSet, basename='user-tickets')
users_router.register(r'traveling-papers', TravelingPaperViewSet, basename='user-papers')

urlpatterns = [
    path('api/', include(router.urls)),
    path('api/', include(users_router.urls)),
]

Security Considerations

  • User can only access their own documents (filtered by user_id in URL)
  • Superusers can access any user's documents via admin interface
  • Files are stored with original filenames - consider adding UUID prefixes for security
  • Implement file size limits and MIME type validation in production
