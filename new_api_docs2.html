BASE URL

1. AUTHENTICATION

Register User

POST/api/users/register/

Request Body:

{
    "email": "john@example.com",
    "password": "securepass123",
    "first_name": "John"
}

Response (201):

{
    "id": 1,
    "email": "john@example.com",
    "first_name": "John"
}

2. USERS

List Users

GET/api/users/users/

Create User

POST/api/users/users/

Request Body:

{
    "email": "jane@example.com",
    "password": "securepass123",
    "first_name": "Jane",
    "middle_name": "Marie",
    "age": 28,
    "date_of_birth": "1996-03-20",
    "nationality": "Filipino",
    "phone_number": "+639123456789",
    "role": "employee",
    "rank_ids": [1, 2],
    "certificate_ids": [1, 3, 5]
}

Get User Detail

GET/api/users/users/{id}/

Update User

PUT/api/users/users/{id}/

Partial Update User

PATCH/api/users/users/{id}/

Delete User

DELETE/api/users/users/{id}/

Filter Users

GET/api/users/filter/?nationality=Egyptian&user_status=active&role=employee

3. USER CERTIFICATES & RANKS

Get User Certificates

GET/api/users/users/{user_id}/certificates/

Add Certificate to User

POST/api/users/users/{user_id}/certificates/add/
{ "certificate_id": 3 }

Remove Certificate from User

DELETE/api/users/users/{user_id}/certificates/{certificate_id}/remove/

Get User Ranks

GET/api/users/users/{user_id}/ranks/

Add Rank to User

POST/api/users/users/{user_id}/ranks/add/
{ "rank_id": 2 }

Remove Rank from User

DELETE/api/users/users/{user_id}/ranks/{rank_id}/remove/

Assign Rank to User

POST/api/users/users/{user_id}/assign-rank/{rank_id}/

Response:

{
    "message": "Rank assigned successfully",
    "assigned_code": "2OFF-001"
}

7. CV SUBMISSIONS

List CV Submissions

GET/api/users/cv-submissions/

Create CV Submission

POST/api/users/cv-submissions/

Content-Type: multipart/form-data

{
    "user": 1,
    "company": 3,
    "position": "Second Officer",
    "cv_file": "<file upload>",
    "cover_letter": "I am interested in the position...",
    "status": "submitted",
    "submitted_date": "2024-06-25"
}

Review CV Submission

PATCH/api/users/cv-submissions/{id}/
{
    "status": "shortlisted",
    "reviewed_date": "2024-06-26",
    "reviewed_by": "James Wilson",
    "rating": 5,
    "feedback": "Excellent candidate, proceed to interview"
}

Filter CV Submissions

GET/api/users/cv-submissions/?status=submitted&user=1&company=1&rating=5

11. AUTHENTICATION HEADERS

For protected endpoints, include the authentication token in the request header:

Authorization: Token <your_token_here>

Or if using JWT:

Authorization: Bearer <your_jwt_token_here>
