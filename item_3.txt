### What Changed

**Replaced the old fragile 3-pass approach** in [document_to_json.py](file:///d:/M SQUARE (MSQ)/CODE SQUARE/Sakr-Manning-Agency-Backend-main/Sakr-Manning-Agency-Backend-main/ai_document/document_to_json.py) with a **single comprehensive Groq LLM call**:

2. **New `_build_comprehensive_prompt()`** — sends the **full document text** (up to 12,000 chars) + **all structured table data** to the LLM with strict extraction rules

5. **`max_tokens` raised from 1024 → 4096** — allows the LLM to output complete data for all sections

### Why It Was Broken Before
- The old `_field()` regex grabbed the **first match** for generic labels like "Name", often pulling the wrong value
- The LLM only received **3,000 chars** of text — no table data at all
- 3 separate LLM calls each with limited context couldn't understand the document as a whole
